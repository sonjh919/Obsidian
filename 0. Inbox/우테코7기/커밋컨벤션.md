- **형식**: `<type>(<scope>): <subject>`
- **타입**:
    
    - `feat`: 새로운 기능
    - `fix`: 버그 수정
    - `docs`: 문서 변경
    - `style`: 코드 스타일 변경 (포맷팅, 세미콜론 누락 등)
    - `refactor`: 코드 리팩토링
    - `test`: 테스트 추가 또는 업데이트
    - `chore`: 유지보수 작업
    
- **스코프**: 영향을 받는 코드 영역을 지정 (예: `$location`, `$browser`).
- **주제**:
    
    - 명령문 현재 시제로 작성 (예: "change" 대신 "changed" 아님).
    - 첫 글자를 대문자로 시작하지 않음.
    - 마침표로 끝내지 않음.
